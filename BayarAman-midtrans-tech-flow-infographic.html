BayarAman + Midtrans Payment Gateway

🎯 Fokus: Payment Otomatis, Dana Tetap Dikontrol BayarAman
Midtrans dipakai untuk menerima pembayaran buyer. BayarAman tetap mengelola status escrow, delivery proof, dispute, release, refund, dan payout seller.
1 FLOW PEMBAYARAN DENGAN MIDTRANS SNAP
1🧾

Seller Buat Transaksi

BayarAman hitung total: amount + fee + kode order.

2⚙️

Server Create Charge

Backend kirim order_id, gross_amount, customer ke Midtrans.

POST /snap/v1/transactions
3🎟️

Dapat Snap Token

Midtrans balikin token / redirect_url untuk halaman bayar.

4💳

Buyer Bayar

VA, QRIS, e-wallet, kartu, atau channel lain aktif di akun.

5🔔

Webhook Masuk

Midtrans POST notification saat status transaksi berubah.

POST /api/midtrans/webhook
6🔐

Verify Signature

Server validasi signature_key, order_id, amount, status.

7🚦

Update Status

settlement/capture → FUNDS_SECURED, expire/cancel/deny → gagal.

8📦

Lanjut Escrow

Seller delivery, buyer review, dispute, release/refund.

⭐ Midtrans menyelesaikan payment collection; BayarAman tetap jadi otak escrow dan operasional trust.

5 STATUS MIDTRANS KE STATUS BAYARAMAN
pendingBuyer sudah masuk flow bayar, belum selesai. BayarAman: WAITING_PAYMENT.
settlementPembayaran sukses. BayarAman: FUNDS_SECURED.
captureKartu sukses/captured. BayarAman: FUNDS_SECURED.
denyPembayaran ditolak. BayarAman: PAYMENT_FAILED.
expireWaktu bayar habis. BayarAman: EXPIRED.
cancelOrder dibatalkan. BayarAman: CANCELLED.
